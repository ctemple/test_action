    │

    ├── ✏️ Edit     精确替换代码

不需要 Python 脚本！ Claude Code CLI 通过 Read/Write/Edit/Bash 工具自主完成任务。 GitHub Actions 只需安装 CLI 并传入 prompt。

2. 前置条件

2.1 DeepSeek API Key

  1. 访问 platform.deepseek.com
  2. 注册并获取 API Key（sk-xxxxxxxx）

2.2 GitHub Personal Access Token (PAT)

  1. 打开 github.com/settings/tokens
  2. Generate new token → Fine-grained token
  3. 选择目标仓库
  4. 权限（必须全部勾选）：
权限级别用途
ContentsRead and write创建分支、提交代码
Pull requestsRead and write创建 PR、提交 Review
IssuesRead and writeIssue 评论、添加标签

2.3 仓库设置

仓库 Settings → Actions → General → Workflow permissions：

  • ✅ Allow GitHub Actions to create and approve pull requests
  • ✅ 选择 Read and write permissions

4. 部署文件

4.1 目录结构

.github/
└── workflows/
    ├── ai-auto-fix.yml          ← AI 自动修复 Issue → PR
    ├── ai-code-review.yml       ← AI 自动审查 PR
    ├── ai-fix-review.yml        ← AI 修复审查意见（闭环）
    └── ai-issue-triage.yml      ← AI 自动分类新 Issue

4.2 四步部署

1 创建 .github/workflows/ 目录

mkdir -p .github/workflows

2 复制以下 4 个文件到该目录：

文件功能触发条件
ai-auto-fix.yml AI 写代码 → PR Issue 打 ai-fix 标签 / 评论 /ai-fix
ai-code-review.yml AI 审查 PR PR 创建/更新 / 评论 /ai-review
ai-fix-review.yml 修复→审查循环 AI Review REQUEST_CHANGES / 评论 /ai-fix-review
ai-issue-triage.yml 自动分类 新 Issue 创建

3 在仓库创建 ai-fix 标签：

gh label create ai-fix --color "7B68EE" --description "AI 自动修复"

4 Push 到 GitHub：

git add .github/workflows/ && git commit -m "feat: add AI workflows" && git push

📄 ai-auto-fix.yml

name: 🤖 AI Auto-Fix Issue

on:
  issues:
    types: [labeled]
  issue_comment:
    types: [created]

permissions:
  contents: write
  pull-requests: write
  issues: write

jobs:
  ai-fix:
    if: |
      (github.event.action == 'labeled' && github.event.label.name == 'ai-fix') ||
      (github.event.action == 'created' &&
       github.event.comment.body == '/ai-fix' &&
       github.event.comment.user.type != 'Bot' &&
       (github.event.comment.author_association == 'OWNER' ||
        github.event.comment.author_association == 'MEMBER' ||
        github.event.comment.author_association == 'COLLABORATOR'))

    runs-on: ubuntu-latest
    timeout-minutes: 30

    env:
      ANTHROPIC_BASE_URL: ${{ secrets.ANTHROPIC_BASE_URL }}
      ANTHROPIC_AUTH_TOKEN: ${{ secrets.ANTHROPIC_AUTH_TOKEN }}
      ANTHROPIC_MODEL: ${{ secrets.ANTHROPIC_MODEL }}
      ANTHROPIC_DEFAULT_HAIKU_MODEL: ${{ secrets.ANTHROPIC_DEFAULT_HAIKU_MODEL }}
      ANTHROPIC_DEFAULT_OPUS_MODEL: ${{ secrets.ANTHROPIC_DEFAULT_OPUS_MODEL }}
      CLAUDE_CODE_DISABLE_1M_CONTEXT: "0"
      GH_TOKEN: ${{ secrets.GH_PAT || github.token }}

    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }

      - uses: actions/setup-node@v4
        with: { node-version: "20" }

      - run: npm install -g @anthropic-ai/claude-code

      - run: |
          git config user.name "🤖 AI Bot"
          git config user.email "ai-bot@github-actions.com"

      - id: issue
        run: |
          if [ "${{ github.event_name }}" = "issues" ]; then
            echo "number=${{ github.event.issue.number }}" >> $GITHUB_OUTPUT
          else
            echo "number=${{ github.event.issue.number }}" >> $GITHUB_OUTPUT
          fi

      - name: 🤖 AI Auto-Fix
        env:
          ISSUE_NUMBER: ${{ steps.issue.outputs.number }}
        run: |
          claude -p "
          Read GitHub issue #$ISSUE_NUMBER using gh issue view.
          Analyze the issue, search the codebase with Glob/Grep,
          read relevant files, implement the fix using Edit/Write.
          Before committing, review your changes with git diff.
          Then: create branch ai/fix-issue-$ISSUE_NUMBER,
          commit with conventional commits message, push,
          create PR with gh pr create, and comment on the issue.
          " --allowedTools "Read,Write,Edit,Bash(git:*),Bash(gh:*),Glob,Grep" \
            --output-format text --verbose

      - if: failure()
        run: |
          gh issue comment ${{ steps.issue.outputs.number }} \
            --repo ${{ github.repository }} \
            --body "❌ AI 修复遇到问题，查看 [日志](${{ github.server_url }}/${{ github.repository }}/actions/runs/${{ github.run_id }})"

📄 ai-code-review.yml

name: 🤖 AI Code Review

on:
  pull_request:
    types: [opened, synchronize, reopened]
  pull_request_review_comment:
    types: [created]

permissions:
  contents: read
  pull-requests: write
  issues: write

concurrency:
  group: ai-review-${{ github.event.pull_request.number || github.event.issue.number }}
  cancel-in-progress: true

jobs:
  ai-review:
    if: |
      github.event_name == 'pull_request' ||
      (github.event_name == 'pull_request_review_comment' &&
       github.event.comment.body == '/ai-review' &&
       (github.event.comment.author_association == 'OWNER' ||
        github.event.comment.author_association == 'MEMBER' ||
        github.event.comment.author_association == 'COLLABORATOR'))

    runs-on: ubuntu-latest
    timeout-minutes: 20

    env:
      ANTHROPIC_BASE_URL: ${{ secrets.ANTHROPIC_BASE_URL }}
      ANTHROPIC_AUTH_TOKEN: ${{ secrets.ANTHROPIC_AUTH_TOKEN }}
      ANTHROPIC_MODEL: ${{ secrets.ANTHROPIC_MODEL }}
      ANTHROPIC_DEFAULT_HAIKU_MODEL: ${{ secrets.ANTHROPIC_DEFAULT_HAIKU_MODEL }}
      ANTHROPIC_DEFAULT_OPUS_MODEL: ${{ secrets.ANTHROPIC_DEFAULT_OPUS_MODEL }}
      CLAUDE_CODE_DISABLE_1M_CONTEXT: "0"
      GH_TOKEN: ${{ github.token }}

    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - uses: actions/setup-node@v4
        with: { node-version: "20" }
      - run: npm install -g @anthropic-ai/claude-code

      - id: pr
        run: |
          if [ "${{ github.event_name }}" = "pull_request" ]; then
            echo "number=${{ github.event.pull_request.number }}" >> $GITHUB_OUTPUT
          else
            echo "number=${{ github.event.issue.number }}" >> $GITHUB_OUTPUT
          fi

      - env:
          PR_NUMBER: ${{ steps.pr.outputs.number }}
        run: |
          claude -p "
          Review PR #$PR_NUMBER. Get the diff with gh pr diff.
          Check for: bugs, security issues, performance problems,
          code style, missing tests, documentation gaps.
          Write review to /tmp/review.md with severity levels
          (CRITICAL/HIGH/MEDIUM/LOW/SUGGESTION).
          Submit using gh pr review with appropriate verdict
          (APPROVE/COMMENT/REQUEST_CHANGES).
          Skip PRs with title starting with 🤖.
          " --allowedTools "Read,Bash(gh:*),Bash(git:*),Write" \
            --output-format text --verbose

      - if: failure()
        run: |
          gh pr comment ${{ steps.pr.outputs.number }} \
            --repo ${{ github.repository }} \
            --body "❌ AI 审查遇到问题，查看 [日志](${{ github.server_url }}/${{ github.repository }}/actions/runs/${{ github.run_id }})"

5. 错误排查

以下是在本仓库实战中遇到的所有错误及其修复方案。

❌ ERROR Author identity unknown

*** Please tell me who you are.
git commit 失败：没有配置 git 用户身份

✅ FIX

在 workflow 中添加 git config：

- run: |
    git config user.name "🤖 AI Bot"
    git config user.email "ai-bot@github-actions.com"

❌ ERROR GitHub Actions is not permitted to create pull requests

GraphQL: GitHub Actions is not permitted to create or approve
pull requests (createPullRequest)

✅ FIX

仓库 Settings → Actions → General → Workflow permissions：
✅ 勾选 "Allow GitHub Actions to create and approve pull requests"

❌ ERROR PR 触发下游 Workflow 需要手动审批

This workflow is awaiting approval from a maintainer

✅ FIX

GITHUB_TOKEN 创建的 PR 被 GitHub 视为外部贡献。
用 GH_PAT（Personal Access Token）代替 GITHUB_TOKEN。

env:
  GH_TOKEN: ${{ secrets.GH_PAT || github.token }}

确保 PAT 权限: Contents: R/W + Pull requests: R/W + Issues: R/W

❌ ERROR Resource not accessible by personal access token

GraphQL: Resource not accessible by personal access token (addComment)

✅ FIX

PAT 缺少 Issues: Read and write 权限。
去 Settings → Tokens 编辑 PAT 添加此权限。

❌ ERROR failed to push some refs — 分支冲突

error: failed to push some refs
hint: Updates were rejected because the tip of your current
branch is behind

✅ FIX

上次运行失败残留的分支导致冲突。push 前清理旧分支：

git push origin --delete ai/fix-issue-N 2>/dev/null || true

（Claude Code CLI 会自动处理此情况，如果用 Python 脚本则需要手动加清理逻辑）

❌ ERROR AI 返回的 JSON 解析失败（缺少外层 {}）

Pass 1 响应解析失败
原始响应: "analysis": "...", "files_to_modify": [...]

✅ FIX

AI 有时输出 JSON 忘记外层 {}。自动补全：

if not stripped.startswith('{'):
    wrapped = '{' + stripped + '}'

（用 Claude Code CLI 后不再有此问题——它直接操作文件而非输出 JSON）

❌ ERROR JSON 字符串内未转义换行符

change_description 字段包含真实换行符 → JSON 解析失败

✅ FIX

方案 A: Prompt 中要求 换行用 \\n 转义
方案 B: 解析前自动转义字符串内的换行符

if in_string and ch == '\n':
    result.append('\\n')

（用 Claude Code CLI 后不再有此问题）

❌ ERROR original_snippet 与文件内容不匹配

AI 生成的 replace 代码片段是猜的，实际文件内容不同 → 修改失败

✅ FIX

两轮生成（Python 脚本方案）：

Pass 1: AI 识别要修改的文件（不生成代码）
Pass 2: 读取实际文件内容 → AI 基于真实内容生成精确补丁

（用 Claude Code CLI 后不再有此问题——它用 Read 工具读取真实文件后再 Edit）

❌ ERROR npm install claude-code 失败 / 太慢

npm ERR! 网络超时或包未找到

✅ FIX

  • 确保 Node.js ≥ 18
  • 使用 actions/setup-node@v4 指定版本 20
  • 首次安装约 1-2 分钟，后续有缓存会更快

❌ ISSUE Actions 列表中出现大量 skipped 记录

Workflow Runs 列表中出现 completed skipped 记录

✅ FIX

Bot 自己的 Issue 评论触发了 issue_comment 事件但条件不匹配。
在条件中添加 Bot 过滤：

github.event.comment.user.type != 'Bot'

6. 验证全链路

  1. 创建测试 Issue： 
    gh issue create --title "[Feature] 简单需求" --label "ai-fix" \
  --body "## 需求描述\n一个简单明确的需求..."
  2. 观察 Actions 标签页：
    • ai-issue-triage 应自动运行 → 添加标签
    • ai-auto-fix 应自动运行 → 创建 PR
    • ai-code-review 应自动运行 → 提交 Review
  3. 验证结果：
    • Issue 有分类标签和自动回复
    • PR 已创建，描述包含 AI 自审摘要
    • PR 有 AI Review（无需手动审批）
